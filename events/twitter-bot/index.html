Rules

Your goal

Develop a unique and creative bot that leverages the Twitter API to send and receive Tweets.

Requirements

The bot you create must meet the following requirements:
  • It should be hosted on a cloud platform, such as Heroku, AWS, an IU server (silo), etc. You can also just host it on your local machine or server, if you'd like. As long as the bot is online and responsive at the time of judging.
  • You can use any programming language you'd like, although Python is recommended.
  • The bot must do something beyond just existing.
  • Code should be uploaded to GitHub or a similar platform before submission and be publicly accessible.
  • To submit, send a link to your bot's profile and the code repository to craig.douglas@gm.com
  • Submit by midnight on Sunday, October 27 (or you can ask for more time)

Winning criteria

  • Wow factor
  • How detailed and thought out is the final product?
  • How substantive is the bot / what does the bot do?
  • Code
    • Readability
    • Cleanliness
    • Style
  • GM-related (bonus points)
